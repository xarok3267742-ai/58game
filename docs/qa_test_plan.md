# QA Test Plan

## Automated
- `./gradlew test`: scoring formula, streak bonus, exact order match, ingredient toggle/clear reducers, correct/wrong order transitions, serve feedback helper, finished-session no-op behavior, timeout loss, target completion, deterministic order generation, endless result behavior, saveable active-session restore helpers including corrupt negative counter sanitization, result feedback helper, level workload copy helper, result primary action, catalog shape, star thresholds and unlock logic.
- `./gradlew lint`: Android static checks.
- `./gradlew assembleDebug`: debug APK build.
- `./gradlew assembleDebugAndroidTest`: instrumentation APK build, including settings progress reset smoke coverage, real DataStore reset coverage and Activity-recreation restore smoke coverage.
- `./gradlew bundleRelease`: release AAB build.
- `./gradlew connectedDebugAndroidTest`: Compose instrumentation smoke tests for onboarding, level completion, wrong-order display with serve feedback, pause/resume, Activity-recreation restore, settings toggles and confirmed local progress reset, plus a real DataStore reset/settings persistence test.
- `python3 scripts/asset_qa.py`: image dimensions and transparent alpha for ingredient sprites, launcher icons, runtime ImageGen bitmaps and real screenshots; writes `build/reports/asset.md` and JSON for the Play handoff.
- `python3 scripts/asset_manifest_qa.py`: verifies accepted runtime/store/ImageGen assets are documented in `docs/asset_manifest.md`, rejected variants are documented, and rejected asset filenames are excluded from release asset paths.
- `python3 scripts/ui_behavior_qa.py`: guards UI behavior wiring for safe sound feedback, platform haptic feedback, reduced-motion timer animation, serve/result feedback wiring, level workload labels, Android Back/background lifecycle opening gameplay pause, configuration-change restore wiring and core gameplay copy, then writes `build/reports/ui_behavior.md` and JSON for the Play handoff.
- `python3 scripts/content_copy_qa.py`: checks visible app/domain copy, Play metadata, listing, privacy and Play answer sheet for required Russian microcopy, length limits and placeholder/dev wording.
- `python3 scripts/accessibility_source_qa.py`: guards custom control semantics, TalkBack labels/states and 48dp+ source-level tap targets.
- `python3 scripts/play_metadata_qa.py`: Play listing length limits, privacy policy terms, feature graphic size and screenshot dimensions; writes `build/reports/play_metadata.md` and JSON for the Play handoff.
- `python3 scripts/play_target_api_qa.py`: verifies `targetSdk >= 35`, `compileSdk >= targetSdk`, release docs reference the current Google Play target API sources and, with `--fetch-policy`, re-checks the official Help Center/Policy pages.
- `python3 scripts/store_visual_quality_qa.py`: checks Play screenshot/feature dimensions, file weight, contrast, color variety and upload-screenshot uniqueness.
- `python3 scripts/store_screenshot_capture_guard_qa.py`: checks that current store screenshots pass the capture candidate validator, a solid black screenshot is rejected, and the capture helper writes screencaps to candidate files before any curated store PNG overwrite.
- `python3 scripts/store_screenshot_freshness_qa.py`: checks whether current UI/gameplay source files are newer than the real store screenshots; use `--strict` in the final upload environment.
- `python3 scripts/store_screenshot_capture_qa.py`: verifies the latest passing real screenshot capture evidence is newer than tracked UI/resource source and checks SHA parity between captured PNGs and current `store/screenshots/`.
- `python3 scripts/play_upload_packet_qa.py`: verifies the Play Console upload packet references required metadata, screenshots, privacy file, signed-AAB warning, strict upload command ordering and policy artifacts; writes `build/reports/play_upload_packet.md` and JSON for the Play handoff.
- `python3 scripts/play_console_forms_qa.py`: verifies Play Console answer coverage for app details, app access, Data safety, permissions, content rating, other app-content declarations, target audience and release path.
- `python3 scripts/play_upload_auth_qa.py`: verifies `SUPPLY_JSON_KEY` all-or-none readiness, service-account JSON shape when set, path outside the repository and absence of service-account credential JSON in source paths.
- `python3 scripts/privacy_data_safety_qa.py`: verifies manifest permissions, target API, backup/data-extraction exclusions, no forbidden SDK/source markers, privacy policy text and Play Console Data safety answers.
- `python3 scripts/privacy_policy_hosting_qa.py`: verifies `store/privacy_policy.html` is static host-ready HTML, writes a checksumed copy under `build/privacy_policy_handoff/`, and in strict mode fetches `SHAWARMA58_PRIVACY_POLICY_URL`.
- `python3 scripts/fastlane_runtime_qa.py`: verifies Gemfile/Gemfile.lock, Ruby configured native headers, available Ruby toolchain manager options, Bundler, `bundle check`, vendor bundle and `bundle exec fastlane --version`; the report includes remediation commands for the upload machine.
- `python3 scripts/completion_audit_qa.py`: verifies the completion audit references required source/docs/assets/metadata and key release blockers, then writes `build/reports/completion_audit.md` and JSON for the Play handoff.
- `python3 scripts/signing_env_qa.py`: validates signing env all-or-none state, keystore path/location when set and absence of keystore-like files in the source workspace.
- `python3 scripts/prepare_upload_keystore.py`: validates upload-keystore setup without storing secrets; with `--generate`, creates the upload key outside the repository after `SHAWARMA58_KEYSTORE*` env vars are set.
- `python3 scripts/upload_operator_runbook_qa.py`: validates `docs/upload_operator_runbook.md` covers required signing, service-account, privacy URL, post-package validation, physical-device and fastlane commands without embedding secrets.
- `python3 scripts/workspace_hygiene_qa.py`: validates `.gitignore` coverage for local caches/secrets, checks for upload-secret-like files outside ignored/generated paths, rejects numbered duplicate report files under `build/reports/`, rejects numbered duplicate artifacts in the transfer root and current handoff root, and verifies the handoff manifest excludes local caches/secrets.
- `python3 scripts/artifact_provenance_qa.py`: validates the built debug APK, release AAB zip structure, merged debug/release manifests, R8 mapping outputs, embedded ProGuard map, package/version/sdk values, forbidden permission strings, secret-like file absence and AAB signing state.
- `python3 scripts/performance_budget_qa.py`: enforces APK/AAB/resource/base-dex/native-lib/dependency budgets and verifies no backend/network/ads/billing/analytics dependency markers.
- `python3 scripts/physical_device_readiness_qa.py`: records whether a non-emulator Android phone is connected for the final physical-device sanity pass; local emulator-only handoffs report `EXTERNAL_BLOCKER`.
- `python3 scripts/pre_upload_blockers_qa.py`: consolidates signing, hosted privacy URL, Play service-account, fastlane runtime and physical-device blockers into one operator summary; use `--strict` before final Play upload.
- `python3 scripts/instrumentation_smoke_qa.py --require-device`: builds the debug app/test APK, waits for the target serial to be online/boot-completed, retries APK install after transient adb daemon/device loss, runs serial-scoped `adb am instrument` and writes `build/reports/instrumentation_smoke.md` plus JSON evidence with structured `testCount`, `failureCount` and `fullSuite` fields.
- `python3 scripts/connected_performance_qa.py --serial <adb-serial>`: waits for the target serial to be online/boot-completed, retries install after transient adb daemon/device loss, runs a serial-scoped instrumentation flow, launches the app, captures `meminfo`, `gfxinfo`, framestats, logcat and crash buffer under `build/performance_connected/`.
- `python3 scripts/capture_store_screenshots.py --serial <adb-serial>`: installs the debug APK through the same hardened install/resolve path as ADB smoke and re-captures the seven real store/QA screenshots into `store/screenshots/` with evidence under `build/store_screenshot_capture/`; text-navigation taps retry until the expected next screen appears, and each screencap is validated as a candidate before it can overwrite the curated store PNG. If the local AVD shuts down mid-flow, `--combine-from-store` creates one full SHA-parity evidence directory from already recaptured real-app PNGs after validating those PNGs.
- `python3 scripts/create_play_handoff.py`: builds a copy-only Play Console handoff directory under `build/play_handoff/` with manifest checksums, root `CHECKSUMS.txt`, signing status, release-gate summary, target API report and the checksumed `privacy/hosting/` publish bundle.
- If a passing connected smoke exists, `python3 scripts/create_play_handoff.py` copies it to `build/play_handoff/shawarma58-v1.0.0/qa/android_smoke/latest/`.
- `python3 scripts/play_handoff_qa.py`: validates the generated handoff manifest, root `CHECKSUMS.txt`, copy-map target coverage against `create_play_handoff.FILES`, file checksums, image sizes, source/SHA parity for every non-generated manifest entry, target API report, privacy hosting bundle SHA parity, release-gate report source/SHA parity and `manifest.releaseGate` summary, asset QA report, UI behavior report, completion-audit report, Play metadata report, artifact provenance report, upload operator runbook report, connected-smoke include/skip state, freshness and source/SHA parity when included, structured instrumentation smoke freshness/full test count against `app/src/androidTest`, optional instrumentation-smoke and connected-performance report source/SHA parity against `build/reports`, connected-performance freshness/raw artifact copies when included, explicit optional instrumentation-smoke and connected-performance include/skip/missing state in `manifest.json`, manifest-based rejected/secret-like file exclusion and operator `NEXT_ACTIONS.md` privacy-hosting plus optional connected-smoke/instrumentation/performance refresh instructions; writes `build/reports/play_handoff_qa.md` and JSON evidence.
- `python3 scripts/update_release_dates.py`: updates release-facing document dates to the current local date before packaging.
- `python3 scripts/play_handoff_archive_qa.py --require-package-report`: validates archive entries, archived `manifest.json` SHA/content parity, SHA sidecar, package-report sidecars, archive sidecar verification command, release-gate summary, optional evidence summary and file checksums against handoff manifest; writes `build/reports/play_handoff_archive_qa.md` and JSON evidence.
- `python3 scripts/play_handoff_secret_scan_qa.py`: scans manifest-listed handoff files, zip entries and transfer sidecars for key-like filenames, private-key blocks and service-account credential JSON, and writes scan-scope counts for handoff/archive/sidecar coverage.
- `python3 scripts/release_freshness_qa.py`: validates current release-document dates, generated report dates including Play metadata evidence, release-gate report freshness, self-checks mandatory handoff QA JSON coverage against `create_play_handoff.FILES`, validates mandatory handoff QA JSON freshness, package status derivation/sidecar parity, exact archive checksum sidecar text, archive/package `manifestSha256` parity and included optional evidence report status/date.
- `python3 scripts/post_package_validation_qa.py`: validates the final non-transfer package state after sidecars are written, including exact transfer file/directory set, sidecar presence/scope, final secret-scan ordering and scan-scope counts, final freshness ordering, post-package report timing after final scan/freshness, package JSON/markdown sidecar parity, package markdown status/SHA/verify-command content, archive SHA/byte parity, checksum sidecar exact text, package verify command, archive/package `manifestSha256` parity, unpacked handoff directory file/SHA/byte parity against `manifest.files`, local handoff/archive QA report status/count parity, local-only post-package report placement, handoff manifest exclusion and package-flow source ordering with no sidecar rewrite after the final scan.
- `python3 scripts/package_release_candidate.py`: reruns secret scan and release freshness after final transfer sidecars are written, then runs post-package validation, so sidecar checks cover the exact files handed off next to the zip without mutating those sidecars afterward.
- `python3 scripts/release_gate.py`: runs local release QA gates, verifies Play upload packet, completion audit, release minify/resource shrinking config, writes the pre-upload blocker summary, writes `build/reports/release_gate.md` plus JSON and reports signing state.
- `python3 scripts/release_gate.py --connected-tests --serial <adb-serial>`: optional full local gate that also runs `python3 scripts/instrumentation_smoke_qa.py --serial <adb-serial> --require-device`.
- `python3 scripts/release_gate.py --connected-performance --serial <adb-serial>`: optional full local gate that also runs `python3 scripts/connected_performance_qa.py --serial <adb-serial>`.

## Manual smoke
1. First launch shows splash then onboarding.
2. Start onboarding and reach main menu.
3. Open level 1, complete one correct order.
4. Submit one wrong order and confirm mistake increments, selected composition clears and `Состав не совпал` is visible.
5. Tap `Пауза`, confirm overlay appears, tap `Продолжить`, then continue the same shift.
6. Complete a level and confirm result stars.
7. Confirm level 2 unlocks.
8. Toggle sound/reduced motion, restart app, confirm settings persist.
9. In settings, tap `Сбросить прогресс`, then `Удалить прогресс`; confirm levels, stars and endless record reset while settings remain.
10. Play endless mode and confirm best score updates.
11. Press back on settings/levels/about/gameplay.
12. Press Home during gameplay, reopen the app and confirm the pause overlay is visible before the timer continues.
13. Test no-internet mode; app should be unaffected.
