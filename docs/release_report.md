# Release Report

## Идея
Выбрана и реализована «Шаурма 58»: offline-first Android casual time-management игра про сбор заказов шаурмы для русскоязычной аудитории.

## Реализовано
- Android проект с Kotlin, Jetpack Compose, Material 3 и DataStore.
- 24 уровня, бесконечная смена, 8 ингредиентов, 4 типа клиентов.
- Core loop: выбор уровня, заказ, ингредиенты, пауза/продолжение, отдача, ошибки, таймер, streak bonus, score, stars, unlock.
- Gameplay now pauses automatically when the app goes to background, so returning from Home/recents does not silently spend shift time.
- Gameplay now restores the active screen/session across Activity recreation through `rememberSaveable`; selected ingredients, timer, score, mistakes, streak and pause state are kept without persisting unfinished runs.
- Serve feedback panel explains `Заказ выдан` or `Состав не совпал` immediately after the player submits an order.
- Splash, onboarding, menu, level select, gameplay, scrollable result with post-shift coaching, settings with two-step local progress reset, about/privacy.
- June 14, 2026 UI refresh differentiated the main screens: storefront hero/action menu, shift-map level select, darker gameplay console/customer ticket, stronger ingredient tiles and receipt-style result layout.
- A later June 14 UI/UX pass further differentiated level select into a `Маршрут 58` route with a current-shift spotlight and changed gameplay ingredients into colored assembly trays with a dark `Линия сборки` rail.
- Current June 14 ImageGen UI/UX pass adds screen-specific portrait backgrounds: route-map art behind level select, a dark prep-station scene behind gameplay and a receipt-counter scene behind result screens. Runtime copies are downscaled to 720x1600 WebP for startup/resource budget, while 1080x2400 PNG sources remain under `store/imagegen_backgrounds/`.
- Level select tiles now show each shift's target order count and duration before launch.
- ImageGen visual set: launcher-safe icon, background, onboarding art, ingredient icons, customer portraits, feature graphic candidate.
- Runtime bitmap assets use optimized WebP in `drawable-nodpi`; ingredient sprites keep transparent alpha and are validated by `scripts/asset_qa.py`.
- Asset manifest QA validates app/store/ImageGen asset documentation, icon concept coverage, round launcher icon coverage and rejected asset exclusion from release paths.
- Release docs, Google Play checklist, QA plan, privacy notes, Play metadata, privacy policy HTML and asset manifest.

- Signing environment QA validates all-or-none upload signing env vars, keystore path/location, alias when `keytool` is available and absence of keystore-like files in the source workspace.
- Upload keystore setup helper validates the upload-key path, keytool availability, env alignment and alias; with `--generate`, it creates the upload keystore outside the repository after signing env vars are set.
- Upload operator runbook QA validates `docs/upload_operator_runbook.md` for final upload-machine env vars, strict signing/auth/privacy/device/runtime commands, post-package validation, signed handoff verification and fastlane validate/upload steps without embedding secrets.
- Workspace hygiene QA validates `.gitignore` coverage for local caches/secrets, scans source paths for upload-secret-like files while pruning ignored/generated roots, rejects stale numbered duplicate files under `build/reports/`, rejects numbered duplicate artifacts in the transfer root and current handoff root, and checks handoff manifest exclusions for local caches/secrets.
- Privacy/data-safety QA validates source and merged release permissions, target API, backup/data-extraction exclusions, forbidden SDK/source markers, privacy policy wording and Play Console answers against the no-data/no-sharing claim.
- Artifact provenance QA validates the built debug APK, release AAB, merged manifests, embedded ProGuard map, release mapping outputs, secret-file absence and signing state; reports are written to `build/reports/artifact_provenance.md` and `build/reports/artifact_provenance.json`.
- Physical device readiness QA records whether a non-emulator Android phone is connected for the final manual sanity pass; emulator-only local handoffs report this as an external blocker.
- R8 mapping outputs are copied into the Play handoff under `deobfuscation/release/` and verified against current Gradle outputs by `scripts/play_handoff_qa.py`.
- Compose instrumentation smoke tests cover onboarding, level workload labels, level completion, serve/result feedback, wrong-order feedback, pause/resume, Android Back pause, Activity-recreation restore, settings toggles and confirmed local progress reset with stable non-visible UI test tags. The wrapper waits for explicit serials to be boot-completed, retries APK install after transient adb daemon/device loss, and writes `build/reports/instrumentation_smoke.md` plus `build/reports/instrumentation_smoke.json` with structured `testCount`, `failureCount`, `classFiltered` and `fullSuite` fields.
- Performance budget QA enforces app size, resource size, base dex, native-lib and dependency allowlist budgets; connected performance QA captures emulator `meminfo`, `gfxinfo` and crash-buffer evidence, and handoff QA verifies the raw evidence is copied into the transfer package when present.
- Store screenshot capture helper can regenerate all seven real screenshots from a connected 1080x2400 emulator/device with `python3 scripts/capture_store_screenshots.py --serial <adb-serial>`; it shares the ADB smoke install hardening (`adb install -r -d`, launchable activity resolution check, clean reinstall of only this debug package on inconsistent package-manager state), and text-navigation taps retry until the expected next screen appears. `--coordinate-mode` captures the same real app flow with text-guarded menu/level navigation plus fixed 1080x2400 ingredient/serve coordinates, foreground checks before each overwrite, QA-only text assertions for `wrong_order`/`endless_result`, and optional temporary disabling/restoring of other third-party packages. Each screencap is first saved as a candidate and validated for dimensions, file weight, contrast and color variety before it can overwrite `store/screenshots/`, so black/headless/foreign captures are rejected before touching curated store PNGs. `--combine-from-store` creates a full SHA-parity evidence directory from the current `store/screenshots/` files after split real-app screencap sessions, and capture QA ignores partial PASS directories.

## Manual Google Play actions
1. Create/upload signing key in Play Console and export env vars:
   `SHAWARMA58_KEYSTORE`, `SHAWARMA58_KEYSTORE_PASSWORD`, `SHAWARMA58_KEY_ALIAS`, `SHAWARMA58_KEY_PASSWORD`.
2. Run `python3 scripts/prepare_upload_keystore.py --generate` if the upload key does not exist yet, then `python3 scripts/prepare_upload_keystore.py --strict`.
3. Re-run `./gradlew bundleRelease` and verify signed AAB with `python3 scripts/release_gate.py --strict-signing`.
4. Upload the recaptured curated Play screenshot pack from `fastlane/metadata/android/ru-RU/images/phoneScreenshots/`. Keep `store/screenshots/shawarma_wrong_order.png` and `store/screenshots/shawarma_endless_result.png` as QA evidence unless a store reviewer wants extra screenshots.
5. Host `store/privacy_policy.html` at a public non-PDF HTTPS URL, run `python3 scripts/privacy_policy_hosting_qa.py --strict --fetch-privacy-url`, and paste it into Play Console.
6. Set `SUPPLY_JSON_KEY` and `SHAWARMA58_PRIVACY_POLICY_URL` if using fastlane.
7. Run `bundle install --path vendor/bundle`, then `python3 scripts/prepare_upload_keystore.py --strict`, then `python3 scripts/play_upload_auth_qa.py --strict`, then `python3 scripts/fastlane_runtime_qa.py --strict`, then `python3 scripts/upload_operator_runbook_qa.py`, then `python3 scripts/pre_upload_blockers_qa.py --strict`, then `python3 scripts/package_release_candidate.py --strict-signing --fetch-privacy-url --fetch-target-api-policy --strict-screenshots --strict-physical-device --strict-pre-upload`, then `python3 scripts/post_package_validation_qa.py`, then `bundle exec fastlane android validate_internal` before any fastlane upload.
